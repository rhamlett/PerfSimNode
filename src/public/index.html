🎮 Simulation Controls

🔥 CPU Stress ℹ️

Spawns child processes running crypto.pbkdf2Sync(). Server stays responsive - work is isolated from the event loop.

📊 Memory Pressure ℹ️

Allocates Buffer blocks filled with random data. Multiple allocations stack - mimics gradual memory leak.

🧵 Event Loop Blocking ⚠️

Runs synchronous crypto in the main thread in repeated chunks. Between chunks, a brief yield lets probe data flush so you see latency spikes in real-time.

What to observe:
  • Latency chart shows sustained high latency during the block
  • Probe dots turn red/orange as each chunk blocks responses
  • Event loop lag spikes in the metrics chart

🐌 Slow Requests ℹ️

Simulates slow HTTP responses. Choose a blocking pattern to observe different resource contention behaviors.

⚠️ Note: Requests exceeding 30 seconds total time (queue + execution) are flagged as critical

❌ Failed Requests ℹ️

Generates HTTP 500 errors by injecting failures into server requests. Errors appear in Azure AppLens and Application Insights.

⚠️ Each request does real work before failing, making errors visible in latency monitoring

💥 Application Crash ⚠️

Terminates the Node.js process. Azure auto-restarts - observe the recovery and new Process ID in Event Log.

⚠️ This will TERMINATE the app!
